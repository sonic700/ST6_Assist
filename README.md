# 街霸6 命中确认辅助练习器(GUI 版)

按出**起手招**(手柄 下+A 或 RT+B,可改)→ 脚本盯对手血条:掉血(**命中,不是被防**)
→ 立刻蜂鸣提示(可选:自动模拟绿冲),然后**冷却 5 秒**不再触发,留给你接连段;
没掉血(被防/挥空)→ 无动静。带图形界面,可选 1P / 2P。

> ⚠ **仅限训练模式离线使用。** 在线对战中使用自动输入违反 Capcom 用户协议。

## 工作方式

**启动即自动绑定游戏**:按进程名 `StreetFighter6.exe` 找到游戏窗口,所有检测/截图/框选
都以游戏画面(客户区)为基准——窗口移动、换分辨率都自动跟随(改尺寸后重采基线即可);
游戏没开时显示"未找到游戏窗口",开了自动接上。

1. **监听手柄输入**(XInput):检测到起手组合(默认 `下+A`、`RT+B`)的瞬间,
   打开一个约 45 帧的「确认窗口」。组合可自定义:界面「起手输入」区点
   **手柄录制** → 4 秒内在手柄上按住你的组合(先按住方向/扳机再按攻击键)即录入,
   选中后可删除,点「保存设置」写回 config(也可直接改 `attack_triggers`);
2. 窗口内对手血条掉血 = 命中 → 提示音 +(可选)自动绿冲 → 冷却 5 秒
   (`cooldown_frames`,接连段期间不会再响);
3. 窗口过了没掉血 = 被防/挥空,日志记一笔,不响;
4. **迸发识别**:掉血的**同时对方斗气槽也突降**(迸发瞬间扣 1 格,先于霸体吃招)
   = 对面在迸发 → **低音双响**(和命中音区分),此时该反迸,自动绿冲也不会注入。
   需要校准 `region_p1_drive` / `region_p2_drive`(对方斗气槽掉气端),
   界面「斗气差」条同血条阈值一样调:普通打一拳看峰值、让假人迸一下看峰值,取中间。

### 自动绿冲(可选,默认关)

通过 **ViGEm 虚拟 Xbox 360 手柄**注入「前前」:脚本把你物理手柄的信号 500Hz
实时透传到虚拟手柄,命中时叠加注入。启用步骤:

1. `pip install vgamepad`(安装时会弹 ViGEmBus 驱动向导,装一次即可);
2. 进游戏后系统里有两个手柄,在 SF6 **控制器设置**里把 **P1 绑到虚拟手柄**;
3. 按 **F7** 试放绿冲验证:角色动了 = 绑对了;没动就换绑另一个手柄;
4. 界面勾上「自动绿冲」。config 的 `combo` 默认是前前,也可以扩成整套连段。

不想用就保持默认关,纯听音自己接——练确认反应本来就该这样。

## 启动

```
python sf6_hit_assist.py
```

或打包成免环境的单文件 exe(见下方「打包 exe」),双击 `SF6HitAssist.exe` 即用。

界面默认置顶,游戏开**无边框窗口**模式即可同屏看到。界面内容:

- **我是哪边**:选 1P(左)或 2P(右),自动决定监控哪条血条(对手那条)和绿冲方向。切边后需重新采基线。
- **状态**:等待起手 → 确认中…(按出起手后) → 冷却中(命中后 5 秒)。
- **起手输入**:当前生效的组合列表,「手柄录制」直接按出来录入,「删除选中」移除。
- **像素差实时条**:绿色=低于阈值,红色=超过(=会触发);黄线是当前阈值位置。
- **阈值滑条**:打中一下看峰值、被防一下看峰值,把黄线调到两者中间。
- **命中后做什么**:提示音 / 自动绿冲(需绑虚拟手柄)两个开关。
- **检测区域实时画面**:这里显示的应该是**对手血条掉血端**,不对就去校准坐标。
- **保存设置**:把选边/阈值/自动绿冲写回 config.json。

## 热键(打游戏时不用切窗口)

键盘全局热键:

| 热键 | 作用 |
|------|------|
| **F9** | 采集基线(站定、对手满血时按一次) |
| **F8** | 启用 / 停用检测 |
| **F7** | 3 秒后试放绿冲(验证虚拟手柄绑定) |
| **F12** | 退出 |

手柄热键(XInput **只读监听**,手不离手柄):按住 **View(视图键,左边小键)** 再按:

| 组合 | 作用 |
|------|------|
| View + **A** | 采集基线(= F9) |
| View + **B** | 启停检测(= F8) |

注:View 键在 SF6 训练模式里本身是「复位位置」,按热键时会顺带复位假人,正好配合采基线。
不想要就把 config 里 `pad_hotkeys` 改 `false`。

## 原理

脚本不读游戏内存、不碰输入,只盯着屏幕上**只有命中才会变化**的小区域——对手血条的掉血端:
普通技被防不掉血,命中必掉血,区分干净。(被防的必杀技有灰色削血,所以请用**普通技**
做确认起手,这正是实战习惯。)采一张基线图后持续对比,像素差超阈值 = 命中 → 响。

## 首次配置(一次性,约 5 分钟)

1. 依赖:`pip install mss numpy`(已装)
2. **框选检测区域**:进训练模式后,在界面用四个框选按钮把
   **1P血条 / 2P血条 / 1P斗气 / 2P斗气** 都框一遍——屏幕会冻结成当前截图,
   鼠标拖一个框即可(Esc 取消),自动写回 config 并提示重新采基线。
   - 血条:框住**血条开始掉血的那一端**(不确定从哪端掉?打一拳看血从哪边缺);
   - 斗气:框住斗气槽**掉气端的最后一格半**(迸发扣最外侧 1 格,框小一点变化越明显);
   - 四个都录好后,选边切换时自动用对应侧的区域,不用再动。
   也可手填 config 的 `region_*` 坐标,`--snap` / `--crop` 可用来核对。
3. **验证**:看界面里两条实时预览(上=血条端,下=斗气槽)对不对。
4. **调阈值**:开检测后打一拳、防一拳,看像素差峰值,黄线调到两者中间,点「保存设置」。

## 日常流程

打开界面 → 选边 → 进训练场站定 → **F9**(或 View+A)采基线 → **F8**(或 View+B)启动
→ 用起手招(下+A / RT+B)开打:命中响一声(冷却 5 秒接连段),被防没动静。
建议假人设「防御:随机」,被防时不该有任何动静,打中才响——这才练得到确认。

起手按了没反应?跑 `python sf6_hit_assist.py --diag`(或 `SF6HitAssist.exe --diag`),
10 秒内按一遍组合,看识别次数,结果同时写到 diag.txt。

## 打包 exe(分发 / 免 Python 环境使用)

双击 [build_exe.bat](build_exe.bat)(或运行 `python build_exe.py`),会自动安装 PyInstaller
并打包出 **`dist\SF6HitAssist.exe`**(单文件,约 24 MB),同时把当前的 config.json 拷进去。

- exe 可拷到**任意目录**双击使用,配置和截图都保存在 exe 同目录;
- 没带 config.json 也行,首次运行会自动生成默认配置;
- 自动绿冲已打包在内,但目标机器要装过 **ViGEmBus 驱动**(装一次即可);
- 命令行功能照常:`SF6HitAssist.exe --snap` / `--crop` / `--diag`(结果以弹窗提示)。

## 注意事项

- 只有前台是游戏时才会触发(防止切出游戏误响):默认认进程名 `StreetFighter6.exe`
  (`window_process`),窗口标题含 `window_title` 也放行;两项都留空则不检查。
- 训练模式假人回血后画面恢复基线,脚本自动重新待命(日志会提示)。
- 检测延迟约 1~3 帧 + 显示器延迟,对绿冲取消确认窗口(命中停顿加持下 20F+)足够。
- 起手招出招慢(窗口过了才打中)就加大 `confirm_window_frames`;
  连段超过 5 秒还在被打断就加大 `cooldown_frames`。
- 自动绿冲没反应:游戏里 P1 没绑到虚拟手柄,按 F7 试放验证。

## 建议练法(配合本目录的 hit-confirm-trainer.html)

1. 先开着提示音打:建立「命中 = 这个声音 = 该接了」的条件反射,听到声马上接绿冲连段;
2. 习惯后假人开随机防御:被防没声就忍住不按,打中有声才出手,刻意练「区分」;
3. 最后关脚本裸练,检验真实确认率。
